# Frontend Directory Structure

The frontend app is rooted at `frontend/`. Most application code lives in `frontend/src/`.

## Layout

```text
frontend/
  src/
    boot/             Quasar boot files
    components/       Reusable Vue components
      site/           Site shell and navigation components
      tools/          Tool-specific reusable panels
    composables/      Reusable Vue stateful logic
    css/              Global SCSS and Quasar variables
    design-system/    Brand tokens and CSS variable helpers
    lib/              API clients and pure/shared data helpers
      http/           Axios request instance and interceptors
    layouts/          Quasar layout components
    pages/            Route-level pages
      tools/          Tool pages such as CSV and Mermaid
    router/           Vue Router configuration
    stores/           Pinia stores
```

Reference files:

- Routes: `frontend/src/router/routes.js`
- Main layout: `frontend/src/layouts/MainLayout.vue`
- Tool pages: `frontend/src/pages/tools/CsvPage.vue`, `frontend/src/pages/tools/MermaidPage.vue`
- API helpers: `frontend/src/lib/csvFiles.js`, `frontend/src/lib/accountSync.js`
- Auth store: `frontend/src/stores/auth.js`

## Pages

Route-level components belong in `src/pages/`. Tool pages belong in `src/pages/tools/` and are routed from `src/router/routes.js`.

Pages may orchestrate several composables, API helpers, and local UI states. If stateful logic becomes reusable or bulky, move it into `src/composables/`.

## Components

Reusable components belong in `src/components/`:

- `components/site/` for navigation, shell, user menu, home-page sections, and theme controls.
- `components/tools/` for reusable tool UI such as `AccountSyncPanel.vue`.

Do not put route-specific components in global component directories unless at least two pages will use them.

## Composables

Reusable stateful logic belongs in `src/composables/` and uses `use*` naming:

- `useAccountSync.js` for generic logged-in sync state.
- `useCsvPreview.js` for browser CSV parsing state.
- `useBrandTheme.js` for theme token selection.

## Lib Modules

Use `src/lib/` for API helpers and shared non-component logic:

- `src/lib/http/` owns the Axios instance and interceptors.
- `auth.js`, `csvFiles.js`, and `accountSync.js` wrap backend endpoints.
- `siteHome.js` and `siteNavigation.js` provide structured content/config.

Keep API URL construction and request details out of components when a lib helper can own them.

## Design System

Global CSS lives in `src/css/app.scss`. Brand tokens and CSS variable helpers live in `src/design-system/`. The Quasar boot file `src/boot/design-system.js` applies stored theme variables on startup.

Prefer extending the token system over hardcoding unrelated palettes in each component.

## Generated And Platform Files

- `src/auto-imports.d.ts` is generated by `unplugin-auto-import`.
- `src-pwa/`, `src-electron/`, and `src-capacitor/` are platform targets.
- `frontend/dist/`, `.quasar/`, and `node_modules/` are generated and ignored.
